Detailed Description

Gadget class.

Library: dbgadget

The IliHTMLReporter class defines a gadget that can be used to generate an HTML document containing the data of a data source. You can specify if you want a table of contents and if the table of contents is positioned after the first page of the document or before the last page of the document. There are four models of document:

  • Classic Table-The data are written into a table. The background is gray and the foreground is black.
  • Table-The data are written into a table. The background is white and the foreground is blue.
  • Classic Form-Each row of the data source is written into a form. The background is gray and the foreground is black.
  • Form-Each row of the data source is written into a form. The background is white and the foreground is blue.
  • Dynamic form-A form displays the current row of the data source of the HTML Document. There is a navigator to change the current row.

Each part of the document is associated with a virtual function and a callback. The calling order depends on the type of document model:

  • If the model type is a classic table or a table, the calling order is the following:
    reportBeginDocument
    reportFirstPageHeading
    reportFirstPageContents
    reportFirstPageFooting
    For documents with the table of contents at the beginning of the document, the following are then called:
    reportTableHeading
    reportTableTitle
    reportTableBeginEntries
    reportTableEntry is called for each entry of the table of contents
    reportTableEndEntries
    reportTableFooting
    reportHeading
    reportBeginRows
    reportRowContent is called for each row of the data source
    reportEndRows
    reportFooting
    For documents with the table of contents at the end of the document, the following are then called:
    reportTableHeading
    reportTableTitle
    reportTableBeginEntries
    reportTableEntry is called for each entry of the table of contents
    reportTableEndEntries
    reportTableFooting
    reportLastPageHeading
    reportLastPageContents
    reportLastPageFooting
    reportEndDocument
  • If the model type is a classic form or a form, the calling order is the following:
    reportBeginDocument
    reportFirstPageHeading
    reportFirstPageContents
    reportFirstPageFooting
    For documents with the table of contents at the beginning of the document, the following are then called:
    reportTableHeading
    reportTableTitle
    reportTableBeginEntries
    reportTableEntry is called for each entry of the table of contents
    reportTableEndEntries
    reportTableFooting
    For each row of the data source, the following functions are called:
    reportHeading
    reportBeginRows
    reportRowContent
    reportEndRows
    reportFooting
    For documents with the table of contents at the end of the document, the following are then called:
    reportTableHeading
    reportTableTitle
    reportTableBeginEntries
    reportTableEntry is called for each entry of the table of contents
    reportTableEndEntries
    reportTableFooting
    reportLastPageHeading
    reportLastPageContents
    reportLastPageFooting
    reportEndDocument
Note
A column is not written if it is not visible.
If you define the callback rowContent, you must call tagDefineLocation function within your row description to insert the row location (used by table of contents). The row location is provided by the rowLocationName property.

About table of contents To create the table of contents, the HTML reporter uses the data of a data source column. For each row, there is a table of contents entry. The entry text is the content of the column for the row.

See also
IliDataGem, IliXML.
