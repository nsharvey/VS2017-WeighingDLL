Condition

The Condition accessor (class IlvConditionAccessor) is used to test a condition on a property when the value is changed.

The first parameter defines a condition operator that is applied to the new value specified in the call to changeValue and to the value specified in the operand parameter. For example, if the value is changed to 10, the operator parameter is >, and the operand is 5, the condition tested is 10 > 5. If the operator is [Operand_value], the condition tested is only the value of the operand parameter (that is, the new value passed to changeValue is ignored).

Depending on the test result, the accessor specified by the Accessor parameter is set to one of two values: Value if True or Value if False. The parameters Operand, Value if True, and Value if False can be either immediate values (such as 1 or "red") or the names of other accessors that will be queried to get the actual values used.

Parameters

  • Operator: The operator used to test the conditions. Can be one of the following: ==, !=, >=, <, <=, or [Operand_value].

  • Operand: The operand value.

  • Value if True: Value to which the output is set if the condition is true (or non 0).

  • Value if False: Value to which the output is set if the condition is false (or 0).

  • Accessor: Prototype value or node attribute that will be set to true or false, depending on the condition.

  • The type of the accessor is undetermined and irrelevant. However, it needs to be compatible with the operand type.
