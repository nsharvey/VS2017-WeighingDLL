Sample: readbm

Reading and Displaying Bitmaps

Loads and displays supported bitmap images.

This example shows you how to use IlvDisplay::readBitmap to load images.
Filenames are given as command line arguments.
Supported image types are:

  • JPEG
  • PNG (including transparency)
  • Windows BMP (all subtypes, compressed or not)
  • Portable Pixmap (ppm, pgm, pbm)
  • TIFF
  • Silicon Graphics RGB
  • WAP Bitmap
Images types are automatically recognized at runtime. If you want to use static linking, then define the compilation macro NOT_USING_MODULES and link with appropiate libraries.

Classes involved:

  • IlvDisplay
  • IlvBitmap
  • IlvBitmapStreamer

Source files:

  • src/readbm.cpp

Sample: createbm

Using Bitmap Data

Shows you how to use bitmap data.

This example shows you how to use bitmap data. Two bitmap data are created:

  • The background bitmap data (ranges from white to black)

The color wheel has a mask created using an IlvBWBitmapData.

Sample: quantize

Using the Color Quantizer

Shows you how to use the color quantizer.

This sample uses the Rogue Wave Views color quantizer to create an indexed image from a true color image.
An image filename can be given as an argument to the executable. The indexed image can have up to 256 colors.
Dithering can be switched on or off.
For best results, it is recommended to use a TrueColor display to run this sample.
When operating on an indexed system, Rogue Wave Views quantizes by default true color images to 64 colors. This allows for multiple TrueColor images to be displayed while retaining accurate visual aspect for each image.

For demonstration purposes, this sample code uses portable GUI elements from the Gadgets package of the Rogue Wave Views suite. Feel free to use these elements in your application if you have a license for Rogue Wave Views or for Rogue Wave Views Controls. This information applies to the classes located in the library ilvgadgt, listed in the README file that you can find in the installation directory.

Classes involved:

  • IlvDisplay
  • IlvBitmapData
  • IlvWUQuantizer
  • IlvPNGStreamer
  • IlvBitmap

Sample: animbm

Using Bitmap Data Operators

Shows you how to use bitmap data operators.

This sample uses the Rogue Wave Views bitmap data to create a rotating view of an image.
An idle procedure updates a bitmap data as fast as possible.
A tile image is loaded and then the foreground bitmap data is generated as a combination of zoom/scale of the tile image.
An additional blur effect can be switched on/off using the 'space' key. This effect uses a background bitmap data. The foreground bitmap data is blended with the background (the blend factor is 150/255), then the background is copied in the foreground data.
Initializing the background with a red color gives a disappearing red flash.

Classes involved:

  • IlvDisplay
  • IlvBitmapData
  • IlvBitmap

Source files:

  • src/animbm.cpp
